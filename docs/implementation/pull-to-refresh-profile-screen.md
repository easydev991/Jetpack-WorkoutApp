# Реализация Pull-to-Refresh на экране профиля

## Обзор

Добавление функционала pull-to-refresh (потянуть для обновления) на экран профиля пользователя для обновления данных профиля и социальной информации (друзья, заявки, дневники и т.д.).

## Зависимости

Используется стандартный `androidx.compose.material3` (включен в проект через Compose BOM 2026.01.00).
Компоненты PullToRefresh (`PullToRefreshBox`, `PullToRefreshState`) доступны в Material3 API и не требуют отдельной библиотеки.

## Примечания по реализации

### Правила разработки

- **Безопасное разворачивание опционалов**: использовать `?`, `?:`, `let`, `checkNotNull` вместо `!!`
- **Логирование**: логи на русском языке через `logger`
- **Обработка ошибок**: не допускать крашей приложения
- **TDD подход**: тесты → логика → UI

### Best Practices из Material 3

- Использовать `PullToRefreshBox` вместо устаревшего `SwipeRefresh`
- Добавить `@OptIn(ExperimentalMaterial3Api::class)` для работы с Pull-to-Refresh API
- Настроить отступы (padding) для корректного отображения индикатора
- Учитывать safe areas и top bar при позиционировании контента
- Компоненты `PullToRefreshBox`, `PullToRefreshState`, `rememberPullToRefreshState()` доступны в основном артефакте `androidx.compose.material3` (отдельная библиотека не требуется)
