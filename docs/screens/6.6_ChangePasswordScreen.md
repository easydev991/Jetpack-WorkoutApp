## Обзор

Экран смены пароля с валидацией (6-32 символа, совпадение), обработкой ошибок и обновлением токена.

## Компоненты

- Три поля ввода паролей (`SWTextField`, `isSecure=true`)
- Кнопка сохранения в bottomBar
- Загрузочный индикатор (`LoadingOverlayView`)
- Отображение ошибок валидации под полями

## Навигация

- **Откуда:** `EditProfileScreen` по кнопке "Изменить пароль"
- **Куда:** Назад в `EditProfileScreen` (при успехе или отмене)

Интеграционные тесты для Use Case могут быть добавлены в будущем по необходимости. В текущем проекте не используется практика интеграционных тестов для Use Case, так как unit-тесты для Repository и Use Case обеспечивают достаточное покрытие.

Токен перегенерируется с новым паролем после успешной смены (как в iOS).

Поля "Новый пароль" и "Подтверждение" показывают ошибки через `isError` и `supportingText`:
- "Минимум 6 символов" — при коротком пароле
- "Пароли должны совпадать" — при несовпадении

Файлы тестов: `ChangePasswordUiStateTest.kt`

## Примечания

- **Токен:** хранится в SecureTokenRepository с Tink шифрованием, обновляется при смене пароля
- **Валидация:** 6-32 символа, совпадение паролей, ошибки показываются в полях
- **API:** `@FormUrlEncoded`, `@Field`, обработка HTTP 400
