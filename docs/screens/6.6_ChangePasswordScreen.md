## Обзор

Экран для смены пароля пользователя. Пользователь должен ввести текущий пароль и новый пароль для подтверждения изменения.

## Назначение

Основная цель и функции экрана:

- Изменение пароля пользователя с подтверждением через текущий пароль
- Проверка требований к паролю (минимум символов, сложность)
- Сравнение нового пароля с подтверждением
- Обработка ошибок при изменении пароля
- Уведомление пользователя об успешном изменении

## Компоненты

- Форма ввода текущего пароля
- Форма ввода нового пароля
- Форма ввода подтверждения нового пароля
- Кнопка сохранения изменений
- Загрузочный индикатор

## Навигация

- Откуда переходят на этот экран:
  - Из экрана редактировани япрофиля (`EditProfileScreen`)

- Куда можно перейти с этого экрана:

  - Назад (пользователь отменяет действие)

### Use Case

```kotlin
class ChangePasswordUseCase(
    private val usersRepository: UsersRepository,
) {
    operator fun invoke(
        current: String,
        new: String,
        confirm: String
    ): Result<Unit> = try {
        // Валидация требований к паролю
        if (current.isEmpty()) {
            return Result.failure(ValidationException("Текущий пароль не может быть пустым"))
        }
        if (new.isEmpty()) {
            return Result.failure(ValidationException("Новый пароль не может быть пустым"))
        }
        if (new.length < MIN_PASSWORD_LENGTH) {
            return Result.failure(ValidationException("Пароль должен содержать минимум $MIN_PASSWORD_LENGTH символов"))
        }
        if (new.length > MAX_PASSWORD_LENGTH) {
            return Result.failure(ValidationException("Пароль не должен превышать $MAX_PASSWORD_LENGTH символов"))
        }
        if (!new.equals(confirm)) {
            return Result.failure(ValidationException("Новый пароль должен совпадать с подтверждением"))
        }

        // Вызов репозитория
        // Токен авторизации добавляется автоматически через TokenInterceptor
        return usersRepository.changePassword(current, new)
    } catch (e: IOException) {
        Result.failure(NetworkException("Не удалось изменить пароль. Проверьте интернет-соединение", e))
    } catch (e: HttpException) {
        when (e.code()) {
            400 -> Result.failure(ValidationException("Неверные данные формы (возможно, неверный текущий пароль)"))
            401 -> Result.failure(AuthException("Требуется авторизация"))
            404 -> Result.failure(AuthException("Пользователь не найден"))
            500 -> Result.failure(ServerException("Ошибка сервера"))
            else -> Result.failure(ServerException("Неизвестная ошибка"))
        }
    }

    companion object {
        private const val MIN_PASSWORD_LENGTH = 6
        private const val MAX_PASSWORD_LENGTH = 32
    }
}
```

### Repository

```kotlin
// UsersRepository.kt
interface UsersRepository {
    suspend fun changePassword(
        current: String,
        new: String
    ): Result<Unit>
}

class UsersRepositoryImpl(
    private val swApi: SWApi,
    private val userPreferencesRepository: UserPreferencesRepository,
) : UsersRepository {
    override suspend fun changePassword(
        current: String,
        new: String
    ): Result<Unit> = try {
        // Проверка авторизации (опционально, так как TokenInterceptor добавляет токен автоматически)
        if (!userPreferencesRepository.isAuthorized()) {
            return Result.failure(AuthException("Требуется авторизация"))
        }

        // Вызов API для изменения пароля
        // Используем поля password (текущий) и new_password (новый)
        val request = ChangePasswordRequest(
            password = current,
            newPassword = new
        )
        val response = swApi.changePassword(request)

        // Проверка статус-кода ответа
        if (response.isSuccessful) {
            Result.success(Unit)
        } else {
            when (response.code()) {
                400 -> Result.failure(ValidationException("Неверные данные формы"))
                401 -> Result.failure(AuthException("Требуется авторизация"))
                404 -> Result.failure(AuthException("Пользователь не найден"))
                500 -> Result.failure(ServerException("Ошибка сервера"))
                else -> Result.failure(ServerException("Неизвестная ошибка"))
            }
        }
    } catch (e: IOException) {
        Result.failure(NetworkException("Не удалось изменить пароль. Проверьте интернет-соединение", e))
    } catch (e: HttpException) {
        // Обработка HTTP исключений
        when (e.code()) {
            400 -> Result.failure(ValidationException("Неверные данные формы"))
            401 -> Result.failure(AuthException("Требуется авторизация"))
            404 -> Result.failure(AuthException("Пользователь не найден"))
            500 -> Result.failure(ServerException("Ошибка сервера"))
            else -> Result.failure(ServerException("Неизвестная ошибка"))
        }
    }
}
```

### ViewModel

```kotlin
// ChangePasswordViewModel.kt
class ChangePasswordViewModel(
    private val changePasswordUseCase: ChangePasswordUseCase,
) : ViewModel() {
    
    // UI State
    private val _uiState = MutableStateFlow<ChangePasswordUiState>(ChangePasswordUiState.Loading)
    val uiState: StateFlow<ChangePasswordUiState> = _uiState.asStateFlow()
    
    // Form fields - public для доступа из UI
    private val _currentPassword = mutableStateOf("")
    val currentPassword: String by _currentPassword
    
    private val _newPassword = mutableStateOf("")
    val newPassword: String by _newPassword
    
    private val _confirmPassword = mutableStateOf("")
    val confirmPassword: String by _confirmPassword
    
    init {
        // Экран готов к работе сразу, без загрузки
        _uiState.value = ChangePasswordUiState.Success(canChangePassword = false)
    }
    
    fun onCurrentPasswordChange(current: String) {
        _currentPassword.value = current
        validateForm()
    }
    
    fun onNewPasswordChange(new: String) {
        _newPassword.value = new
        validateForm()
    }
    
    fun onConfirmPasswordChange(confirm: String) {
        _confirmPassword.value = confirm
        validateForm()
    }
    
    fun onChangePassword() {
        viewModelScope.launch {
            _uiState.value = ChangePasswordUiState.Loading
            
            val result = changePasswordUseCase(
                current = currentPassword,
                new = newPassword
            )
            
            when (result) {
                is Success -> {
                    _uiState.value = ChangePasswordUiState.Success(canChangePassword = true)
                    // Очистить поля
                    _currentPassword.value = ""
                    _newPassword.value = ""
                    _confirmPassword.value = ""
                }
                is Failure -> {
                    _uiState.value = ChangePasswordUiState.Error(result.exceptionOrNull()?.message ?: "Ошибка изменения пароля")
                }
            }
        }
    }
    
    private fun validateForm() {
        // Текущий пароль не может быть пустым
        val isCurrentPasswordValid = currentPassword.isNotEmpty()
        
        // Новый пароль не может быть пустым
        val isNewPasswordValid = newPassword.isNotEmpty()
        
        // Подтверждение должно совпадать с новым паролем
        val isConfirmValid = confirmPassword.isNotEmpty() && 
                          confirmPassword == newPassword
        
        // Проверка длины пароля (минимум 6, максимум 32 символа)
        val isPasswordLengthValid = newPassword.length in 6..32
        
        val canChangePassword = isCurrentPasswordValid && isNewPasswordValid && 
                              isConfirmValid && isPasswordLengthValid
        _uiState.value = if (canChangePassword) {
            ChangePasswordUiState.Success(canChangePassword = true)
        } else {
            ChangePasswordUiState.Success(canChangePassword = false)
        }
    }
}
```

```kotlin
// ChangePasswordScreen.kt
@OptIn(ExperimentalMaterial3Api::class)
@Composable
fun ChangePasswordScreen(
    viewModel: ChangePasswordViewModel,
    onNavigateToProfile: () -> Unit,
    onNavigateBack: () -> Unit,
) {
    val uiState by viewModel.uiState.collectAsState()
    
    Scaffold(
        topBar = {
            TopAppBar(
                title = stringResource(R.string.change_password),
                navigationIcon = {
                    Icon(
                        imageVector = Icons.Default.ArrowBack,
                        contentDescription = "Назад"
                    )
                },
                actions = {
                    IconButton(onClick = { onNavigateBack() })
                }
            )
        }
    ) { padding ->
        Column(
            modifier = Modifier
                .padding(padding)
                .fillMaxSize(),
            verticalArrangement = Arrangement.spacedBy(16.dp)
        ) {
            when (uiState) {
                is ChangePasswordUiState.Loading -> {
                    LoadingOverlayView(
                        isLoading = true,
                        message = "Загрузка..."

                        SWTextField(
                            value = viewModel.newPassword,
                            onValueChange = { viewModel.onNewPasswordChange(it) },
                            placeholder = stringResource(R.string.enter_new_password),
                            isSecure = true,
                            isError = viewModel.newPassword.isEmpty() && 
                                       uiState is ChangePasswordUiState.Success
                        )
                    }
                    
                    // Подтверждение нового пароля
                    SectionView(header = stringResource(R.string.confirm_new_password)) {
                        SWTextField(
                            value = viewModel.confirmPassword,
                            onValueChange = { viewModel.onConfirmPasswordChange(it) },
                            placeholder = stringResource(R.string.confirm_password),
                            isSecure = true,
                            isError = viewModel.confirmPassword.isEmpty() && 
                                       uiState is ChangePasswordUiState.Success
                        )
                    }
                    
                    Spacer(modifier = Modifier.height(24.dp))
                    
                    // Кнопка сохранения изменений
                    SWButton(
                        text = stringResource(R.string.save_changes),
                        onClick = { viewModel.onChangePassword() },
                        modifier = Modifier.fillMaxWidth(),
                        enabled = uiState.canChangePassword,
                        style = SWButtonStyle(mode = .filled, size = .large)
                    )
                }
                is ChangePasswordUiState.Error -> {
                    Column(
                        modifier = Modifier.fillMaxSize(),
                        horizontalAlignment = Alignment.CenterHorizontally,
                        verticalArrangement = Arrangement.Center
                    ) {
                        Text(
                            text = uiState.message,
                            color = MaterialTheme.colorScheme.error,
                            style = MaterialTheme.typography.bodyMedium
                        )
                        
                        Spacer(modifier = Modifier.height(16.dp))
                        
                        Button(
                            onClick = { onNavigateBack() },
                            text = stringResource(R.string.back)
                        )
                    }
                }
            }
        }
    )
}
```

## Примечания

- **Безопасность паролей**: все поля ввода паролей используют `isSecure: true` для скрытия ввода текста
- **Токен авторизации**: хранится в SecureTokenRepository с шифрованием через Tink (AES-128-GCM-HKDF)
- **TokenInterceptor**: автоматически добавляет токен в заголовок `Authorization: Basic {token}` для всех HTTP запросов
- **Изменение пароля не влияет на токен**: при успешной смене пароля токен остаётся валидным, повторная авторизация не требуется
- **Валидация**: реализована в ViewModel с проверкой длины (минимум 6, максимум 32 символа) и совпадения с подтверждением
- **Подтверждение пароля**: это валидация на стороне приложения, сервер не требует поле `confirm`
- **Обработка ошибок**: реализована в Use Case с информативными сообщениями на русском языке
- **Консистентность с iOS-приложением**: UI соответствует подходу iOS (формы ввода, кнопка сохранения)
- **Material Design 3**: использование Material 3 для всех компонентов
- **Навигация**: при успешном изменении переходит на `EditProfileScreen`, при отмене возвращается назад
