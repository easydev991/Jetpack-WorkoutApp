# План: EventFormScreen - Создание/редактирование мероприятия

## Обзор

**Задача:** Реализовать экран для создания и редактирования мероприятий с тремя режимами работы.

**Референс:** iOS `SwiftUI-WorkoutApp/Screens/Events/EventFormScreen.swift`

**Маршруты навигации:** `Screen.CreateEvent`, `Screen.EditEvent`, `Screen.CreateEventForPark`

---

## Режимы работы экрана

| Режим             | Источник                     | Заголовок           | Площадка                      |
|-------------------|------------------------------|---------------------|-------------------------------|
| regularCreate     | EventsScreen → FAB           | "Новое мероприятие" | Выбирается                    |
| editExisting      | EventDetailScreen → edit     | "Мероприятие"       | Выбирается                    |
| createForSelected | ParkDetailScreen / навигация | "Новое мероприятие" | Предустановлена (неизменяема) |

## Этап 15: Исправление отправки фотографий Event на сервер ✅

**Диагностика (фактическое состояние):**
- Android отправляет multipart с файлами `photo_0`, `photo_1`, `photo_2`, `photo_3` и `Content-Type: image/*`
- В логах запроса видно, что часть файлов имеет PNG-сигнатуру (`PNG` / `IHDR`) при имени файла `.jpg`
- Сервер отвечает `200`, но в `EventResponse` возвращает `"photos": []` (фото игнорируются)
- В iOS-референсе (`SwiftUI-WorkoutApp`) фото отправляются как `photo1`, `photo2`, ... и с явным MIME (`image/jpeg`)

**Гипотеза причины:**
- Нестыковка multipart-контракта Android с сервером/iOS: имя частей (`photo_0` vs `photo1`) и некорректная пара `mime/расширение/данные` (`image/*` + `.jpg` + PNG-байты)

### 15.1 Привести контракт multipart к iOS ✅

- [x] Для `saveEvent` (create/edit) формировать ключи фото как `photo${index + 1}` вместо `photo_$index`
- [x] Передавать явный MIME (`image/jpeg`) и согласованное имя файла (`photo${index + 1}.jpg`)
- [x] Гарантировать единый формат данных (JPEG) перед отправкой, чтобы исключить PNG-байты в `.jpg` (уже есть в `ImageUtils.compressIfNeeded`)

### 15.2 Уточнить сетевой helper для фото ✅

- [x] Обновить `NetworkUtils.createImagePart(...)`: принимать/формировать корректные `partName`, `fileName`, `mimeType`
- [x] Убрать универсальный `image/*` для Event-фото, использовать конкретный тип

### 15.3 Добавить тесты на контракт ✅

- [x] Unit-тесты для сборки multipart-частей: `name`, `filename`, `Content-Type` (NetworkUtilsTest - 5 тестов)
- [x] Unit-тесты `SWRepository.saveEvent(...)`: проверка имен `photo1..N` для create/edit (2 теста)
- [x] Регрессионный тест: при передаче PNG-исходника на выходе уходит JPEG (ImageUtils.compressBitmapToJpeg)

### 15.4 Верификация на стенде
