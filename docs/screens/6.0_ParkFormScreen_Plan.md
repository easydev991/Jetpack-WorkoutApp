# План: ParkFormScreen - Создание/редактирование площадки

## Обзор

**Задача:** Реализовать экран для создания и редактирования площадок (парков) с двумя режимами работы.

**Референс:**
- Android: `EventFormScreen.kt` (аналогичная форма)
- iOS: `SwiftUI-WorkoutApp/Screens/Parks/ParkFormScreen.swift` + `SWModels/ParkForm.swift` + `ParkFormTests.swift`

**Статус:** Этапы 1-4 завершены. Этап 5 реализован частично. P1 blockers исправлены, P0 (create-flow с координатами) остаётся как зависимость от Stage 6.

**Ключевое отличие от EventFormScreen:**
- Вместо `ParkSelector` (один селектор площадки) -> два селектора **типа** и **размера** площадки
- Оба селектора реализованы как **RadioButton** (по аналогии с `RegisterGenderSection`)
- Логика `canSave` и валидация формы должны полностью совпадать с iOS-референсом

## TDD-стратегия (обязательно)

Для каждого этапа использовать цикл:
1. **RED**: сначала написать/обновить тесты и зафиксировать, что они падают на текущей реализации.
2. **GREEN**: добавить минимальную реализацию, чтобы тесты стали зелеными.
3. **REFACTOR**: улучшить код без изменения поведения и повторно прогнать тесты/lint.

Правило для этой задачи: **ни одного production-изменения без соответствующего теста, который был написан до реализации**.
