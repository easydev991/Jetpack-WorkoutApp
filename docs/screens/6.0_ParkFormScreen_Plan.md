# План: ParkFormScreen - Создание/редактирование площадки

## Обзор

**Задача:** Реализовать экран для создания и редактирования площадок (парков) с двумя режимами работы.

**Референс:** Android `EventFormScreen.kt`; iOS `ParkFormScreen.swift` + `ParkForm.swift`

**Статус:** Этапы 1-8 завершены.

## Основные компоненты

### iOS ParkForm (ParkGrade/ParkSize)

| Поле | Тип | Описание |
|------|-----|----------|
| `address` | String | Адрес площадки |
| `latitude` | String | Широта |
| `longitude` | String | Долгота |
| `cityId` | Int? | ID города |
| `typeId` | Int | Тип площадки (ParkGrade) |
| `sizeId` | Int | Размер площадки (ParkSize) |
| `photosCount` | Int | Количество существующих фото |
| `newMediaFiles` | [MediaFile] | Новые фото |

### Android ParkType / ParkSize

| ParkType | rawValue | R.string |
|----------|----------|----------|
| SOVIET | 1 | `soviet_park` |
| MODERN | 2 | `modern_park` |
| COLLARS | 3 | `collars_park` |
| LEGENDARY | 6 | `legendary_park` |

| ParkSize | rawValue | R.string |
|----------|----------|----------|
| SMALL | 1 | `small_park` |
| MEDIUM | 2 | `medium_park` |
| LARGE | 3 | `large_park` |

## Этап 7 (TDD): Синхронизация кэша addedParks ✅

**Проблема:** `SWRepository.savePark()`/`deletePark()` не обновляют локальный кэш `UserEntity.addedParks`.

**Решение:** Атомарные операции в `UserDao` + синхронизация кэша в `SWRepository`.

## Этап 8 (TDD): Геокодирование с русской локализацией ✅

**Проблема:** `GeocodingServiceImpl` использует дефолтную локаль устройства → `countryName`/`locality` на английском, но `CountriesRepository` содержит только русские названия.

**Решение:** `Geocoder` создаётся с `Locale("ru", "RU")`.

| Задача | Статус |
|--------|--------|
| `geocoderProvider` signature: `(Context) → (Context, Locale) → Geocoder` | ✅ |
| Lazy init: `geocoderProvider(context, Locale("ru", "RU"))` | ✅ |
| Тест `reverseGeocode_whenSuccess_returnsResultWithRussianLocale` | ✅ |

**Тестов:** 5 (`GeocodingServiceImplTest`), **всего:** 1564 unit tests
