### Логика отображения кнопки создания записей

Кнопка отображается на основе логики `JournalAccess.canCreateEntry(...)`, аналогично iOS-проекту:

**Правила отображения:**

- `JournalAccess.FRIENDS`: кнопка видна друзьям владельца дневника И самому владельцу
- `JournalAccess.NOBODY`: кнопка видна только владельцу дневника

#### 1. Domain Layer: Расширение JournalAccess ✅ (существует, нужно добавить метод)

**Файл:** `app/src/main/java/com/swparks/ui/model/JournalAccess.kt`

**Задачи:**
- Добавить extension функцию `canCreateEntry(...)` в `JournalAccess.kt`:

```kotlin
/**
 * Проверяет возможность создания записи в дневнике
 *
 * @param journalOwnerId ID владельца дневника
 * @param mainUserId ID авторизованного пользователя (null если не авторизован)
 * @param mainUserFriendsIds Список ID друзей авторизованного пользователя
 * @return true - можно создать запись, false - нельзя
 */
fun JournalAccess.canCreateEntry(
    journalOwnerId: Long,
    mainUserId: Long?,
    mainUserFriendsIds: List<Long>
): Boolean {
    if (mainUserId == null) return false
    val isOwner = journalOwnerId == mainUserId
    // Проверяем, является ли владелец дневника другом текущего пользователя
    val isFriend = mainUserFriendsIds.contains(journalOwnerId)

    return when (this) {
        JournalAccess.ALL -> true // уже проверили, что mainUserId != null
        JournalAccess.FRIENDS -> isFriend || isOwner // владелец всегда может создавать записи
        JournalAccess.NOBODY -> isOwner
    }
}
```

**Тестирование:**
- 7 unit-тестов:
  1. `ALL` + авторизован → `true`
  2. `ALL` + не авторизован → `false`
  3. `FRIENDS` + друг → `true`
  4. `FRIENDS` + не друг → `false`
  5. `FRIENDS` + владелец (не друг) → `true` (владелец всегда может)
  6. `NOBODY` + владелец → `true`
  7. `NOBODY` + не владелец → `false`

**Пример реализации `computeCanCreateEntry()`:**

```kotlin
private fun computeCanCreateEntry(): Flow<Boolean> {
    val commentAccessType = savedStateHandle.get<String>("commentAccess")
        ?.let { JournalAccess.valueOf(it) }
        ?: JournalAccess.NOBODY // по умолчанию самый строгий режим

    return combine(
        currentUserId,
        friendsFlow.map { friends -> friends.map { it.id } }
    ) { userId, friendsIds ->
        commentAccessType.canCreateEntry(
            journalOwnerId = this.userId, // из savedStateHandle
            mainUserId = userId,
            mainUserFriendsIds = friendsIds
        )
    }
}

// В init блоке:
viewModelScope.launch {
    computeCanCreateEntry().collect { canCreate ->
        _uiState.update { it.copy(canCreateEntry = canCreate) }
    }
}
```

**Пример реализации `canEditEntry()`:**

```kotlin
override fun canEditEntry(entry: JournalEntry): Boolean {
    return currentUserId.value == entry.userId
}
```
