## Описание экрана

Универсальный экран ввода текста для создания/редактирования комментариев и записей в дневнике. Открывается как bottom sheet, который нельзя закрыть жестами (только через кнопку X), содержит текстовое поле SWTextEditor и кнопку "Отправить". Поддерживает 6 сценариев через Mode enum с разными заголовками и поведением.

## Референсы

- iOS-версия: `SwiftUI-WorkoutApp/Screens/Common/TextEntryScreen.swift`
- iOS-версия: `SwiftUI-WorkoutApp/Screens/Common/SendMessageScreen.swift`
- UI компоненты: `SWTextEditor.kt`, `SWButton.kt`
- Bottom Sheet: `LoginSheetHost.kt` - референс паттерна для управления состоянием bottom sheet (состояние `showLoginSheet` в RootScreen.kt)
- Безопасные зоны: `LoginSheetHost.kt` - паттерн для modal bottom sheet с запретом закрытия жестами

## Правила работы с размерами и отступами

**Важно:** Для кастомных значений использовать следующие правила:
- Не создавать новые dimens для значений, близких к существующим (например, 26dp → использовать `spacing_large` = 24dp)
- Для специфических значений без аналогов (например, высота текстового поля) использовать обычные значения без dimens: `200.dp`
- Для всех остальных случаев использовать существующие значения из `app/src/main/res/values/dimens.xml`

**Доступные spacings:**
- `spacing_micro` = 2dp
- `spacing_xxsmall` = 4dp
- `spacing_xxsmall_plus` = 6dp
- `spacing_xsmall` = 8dp
- `spacing_xsmall_plus` = 10dp
- `spacing_small` = 12dp
- `spacing_small_plus` = 14dp
- `spacing_regular` = 16dp
- `spacing_medium` = 20dp
- `spacing_medium_plus` = 22dp
- `spacing_large` = 24dp
- `spacing_xlarge` = 32dp

## Этап 1: UI Model - TextEntryMode и EditInfo

### Задачи

**1. Создать enum TextEntryMode в `ui/model/`:**

**Примечание:** `TextEntryOption` уже существует в репозитории и используется для определения типа объекта (Park/Event/Journal). `TextEntryMode` нужен для UI-слоя, чтобы определять режим работы экрана (заголовок, placeholder, валидация).

```kotlin
sealed class TextEntryMode {
    // Новые записи
    data class NewForPark(val parkId: Long) : TextEntryMode()
    data class NewForEvent(val eventId: Long) : TextEntryMode()
    data class NewForJournal(val ownerId: Long, val journalId: Long) : TextEntryMode()

    // Редактирование
    data class EditPark(val editInfo: EditInfo) : TextEntryMode()
    data class EditEvent(val editInfo: EditInfo) : TextEntryMode()
    data class EditJournalEntry(val ownerId: Long, val editInfo: EditInfo) : TextEntryMode()
}

data class EditInfo(
    val parentObjectId: Long,
    val entryId: Long,
    val oldEntry: String
)
```

**2. Создать интерфейс для получения заголовка и placeholder:**

**Примечание:** Возвращаем `Int` (ID строкового ресурса), а не `String`, так как Compose ожидает ID для `stringResource()`.

```kotlin
@Composable
fun TextEntryMode.getTitle(): Int = when (this) {
    is NewForPark, is NewForEvent -> R.string.new_comment_title
    is NewForJournal -> R.string.new_entry_title
    is EditPark, is EditEvent -> R.string.edit_comment_title
    is EditJournalEntry -> R.string.edit_entry_title
}

@Composable
fun TextEntryMode.getPlaceholder(): Int? = when (this) {
    is NewForJournal -> R.string.new_entry_placeholder
    else -> null
}
```

### Задачи

**Примечание:** Паттерн аналогичен `LoginScreen` - обычный Composable без `ModalBottomSheet`. Вся логика управления sheet находится в `TextEntrySheetHost`.

**1. Создать `TextEntryScreen` composable:**

**Важно:** Использовать существующие компоненты:
- `SWTextEditor` - текстовое поле
- `SWButton` - кнопка отправки
- `LoadingOverlayView` - оверлей загрузки
- `CenterAlignedTopAppBar` - топ-бар с кнопкой закрытия

**Важно:** Использовать существующие dimens вместо hardcoded значений:
- `16.dp` → `dimensionResource(R.dimen.spacing_regular)`

**Важно:** `TextEntryScreen` - "глупый" UI-компонент, который НЕ должен знать о ViewModel events. Вся обработка событий выполняется в `TextEntrySheetHost`.

**Важно:** Сигнатуры компонентов проверены и соответствуют фактической реализации:
- `SWButton(config: ButtonConfig)` - использует `ButtonConfig` для конфигурации
- `SWTextEditor(text, onTextChange, labelID, modifier, enabled)` - использует `labelID` для placeholder

```kotlin
@OptIn(ExperimentalMaterial3Api::class)
@Composable
fun TextEntryScreen(
    modifier: Modifier = Modifier,
    viewModel: ITextEntryViewModel,
    onDismiss: () -> Unit
) {
    val uiState by viewModel.uiState.collectAsState()
    val scope = rememberCoroutineScope()

    Scaffold(
        modifier = modifier,
        topBar = {
            CenterAlignedTopAppBar(
                title = {
                    Text(
                        text = stringResource(uiState.mode.getTitle())
                    )
                },
                navigationIcon = {
                    IconButton(
                        onClick = onDismiss,
                        enabled = !uiState.isLoading
                    ) {
                        Icon(
                            imageVector = Icons.Default.Close,
                            contentDescription = stringResource(R.string.close_button_content_description)
                        )
                    }
                },
                colors = TopAppBarDefaults.centerAlignedTopAppBarColors(
                    containerColor = MaterialTheme.colorScheme.surface
                )
            )
        }
    ) { paddingValues ->
        Box(modifier = Modifier.fillMaxSize()) {
            Column(
                modifier = Modifier
                    .fillMaxSize()
                    .padding(paddingValues)
                    .padding(horizontal = dimensionResource(R.dimen.spacing_regular))
                    .padding(bottom = dimensionResource(R.dimen.spacing_regular))
            ) {
                Spacer(modifier = Modifier.height(dimensionResource(R.dimen.spacing_regular)))

                // Текстовое поле
                SWTextEditor(
                    text = uiState.text,
                    onTextChange = viewModel::onTextChanged,
                    labelID = uiState.mode.getPlaceholder()
                        ?: R.string.new_entry_placeholder,
                    modifier = Modifier
                        .fillMaxWidth()
                        .defaultMinSize(minHeight = 200.dp),
                    enabled = !uiState.isLoading
                )

                Spacer(modifier = Modifier.weight(1f))

                // Кнопка отправки
                SWButton(
                    config = ButtonConfig(
                        modifier = Modifier.fillMaxWidth(),
                        size = SWButtonSize.LARGE,
                        mode = SWButtonMode.FILLED,
                        text = stringResource(R.string.send_button_text),
                        enabled = uiState.isSendEnabled && !uiState.isLoading,
                        onClick = {
                            scope.launch { viewModel.onSend() }
                        }
                    )
                )
            }

            // Loading overlay
            if (uiState.isLoading) {
                LoadingOverlayView()
            }
        }
    }
}
```

**Критерий завершения:** Экран создан, кнопка X работает, кнопка Отправить работает

**2. Создать `TextEntrySheetHost` (аналогично LoginSheetHost):**

**Примечание:** `TextEntrySheetHost` содержит всю логику управления bottom sheet и ВСЮ обработку событий ViewModel (как Success, так и Error).

```kotlin
@OptIn(ExperimentalMaterial3Api::class)
@Composable
fun TextEntrySheetHost(
    show: Boolean,
    mode: TextEntryMode,
    onDismissed: () -> Unit,
    onSendSuccess: () -> Unit = {}
) {
    var allowHide by remember { mutableStateOf(false) }
    val scope = rememberCoroutineScope()

    // Snackbar host для сообщений
    val snackbarHostState = remember { SnackbarHostState() }

    val appContainer = remember {
        (LocalContext.current.applicationContext as JetpackWorkoutApplication).container
    }

    val viewModel: ITextEntryViewModel = remember(mode) {
        appContainer.textEntryViewModelFactory(mode)
    }
    val uiState by viewModel.uiState.collectAsState()

    // Сбрасываем состояние при каждом открытии sheet
    LaunchedEffect(show) {
        if (show) {
            viewModel.resetState()
        }
    }

    // Обработка ВСЕХ событий ViewModel (Success и Error)
    LaunchedEffect(Unit) {
        viewModel.events.collect { event ->
            when (event) {
                is TextEntryEvent.Success -> {
                    // Логика закрытия sheet и внешний коллбэк
                    scope.launch {
                        allowHide = true
                        sheetState.hide()
                        allowHide = false
                        onSendSuccess() // Внешний коллбэк для родительского компонента
                    }
                }
                is TextEntryEvent.Error -> {
                    snackbarHostState.showSnackbar(event.message)
                }
            }
        }
    }

    val sheetState = rememberModalBottomSheetState(
        skipPartiallyExpanded = true,
        confirmValueChange = { newValue ->
            if (newValue == SheetValue.Hidden) allowHide else true
        }
    )

    if (show) {
        ModalBottomSheet(
            onDismissRequest = {
                // Игнорируем тап вне sheet / системный dismiss
            },
            sheetState = sheetState,
            dragHandle = {},
            properties = ModalBottomSheetProperties(
                shouldDismissOnBackPress = false
            )
        ) {
            Box(modifier = Modifier.fillMaxSize()) {
                TextEntryScreen(
                    viewModel = viewModel,
                    onDismiss = {
                        if (uiState.isLoading) return@TextEntryScreen
                        scope.launch {
                            allowHide = true
                            sheetState.hide()
                            allowHide = false
                            onDismissed()
                        }
                    }
                )

                // Snackbar для ошибок
                SnackbarHost(
                    hostState = snackbarHostState,
                    modifier = Modifier.align(Alignment.BottomCenter)
                )
            }
        }
    }
}
```

**Критерий завершения:** Навигация настроена, экран открывается из нужных мест, паттерн соответствует `showLoginSheet`

## Архитектурные решения

### TextEntryMode vs TextEntryOption

- `TextEntryOption` - уже существует в репозитории, используется для определения типа объекта (Park/Event/Journal)
- `TextEntryMode` - нужно создать для UI-слоя, определяет режим работы экрана (заголовок, placeholder, валидация)
- `TextEntryMode` конвертируется в `TextEntryOption` при вызове Use Case

### API и Repository

- API endpoints и Repository методы уже реализованы:
  - `SWApi.addCommentToPark`, `addCommentToEvent`, `saveJournalEntry` - для добавления
  - `SWApi.editParkComment`, `editEventComment`, `editJournalEntry` - для редактирования
  - `SWRepository.addComment(option: TextEntryOption, comment: String)` - универсальный метод
  - `SWRepository.editComment(option: TextEntryOption, commentId: Long, newComment: String)` - универсальный метод
- API принимает простые строки (без DTOs)

### Use Case слой

- `TextEntryUseCase` - делегирует вызовы Repository, используя `TextEntryOption`
- Преобразует `TextEntryMode` в соответствующий `TextEntryOption`
- Добавляет логирование операций

### Mode enum

- Использовать sealed class для возможности хранения данных (ID, EditInfo)
- Заголовок и placeholder получать через extension функции
- Режимы для создания и редактирования разделены

### Bottom Sheet

**Рефере́нс паттерна:** `LoginSheetHost.kt` + `LoginScreen.kt`

- Использовать `ModalBottomSheet` из Material 3 в `TextEntrySheetHost`
- `TextEntryScreen` - обычный Composable без `ModalBottomSheet`, использует `Scaffold` с `CenterAlignedTopAppBar`
- `skipPartiallyExpanded = true` для полного развертывания
- Состояние (`show`) хранится в родительском компоненте
- Callback для закрытия (`onDismissed`) передается наверх через параметр
- Callback для успешной отправки (`onSendSuccess`) передается наверх через параметр
- Запрет закрытия жестами через `onDismissRequest = { }`
- Разрешить закрытие только через кнопку X (с проверкой `!uiState.isLoading`)
- Snackbar для ошибок находится на уровне `TextEntrySheetHost`, а не внутри `TextEntryScreen`
- **Важно:** ВСЯ обработка событий ViewModel (`Success` и `Error`) выполняется в `TextEntrySheetHost`, `TextEntryScreen` не подписывается на `viewModel.events` (это предотвращает двойную подписку на Flow)

**Пример использования (аналогично showLoginSheet в RootScreen.kt):**

```kotlin

// При открытии:

    onShowTextEntrySheet = { showTextEntrySheet = true }
)

// Sheet поверх навигации:
TextEntrySheetHost(
    show = showTextEntrySheet,
    mode = TextEntryMode.NewForJournal(userId, journalId),
    onDismissed = { showTextEntrySheet = false },
    onSendSuccess = {
        showTextEntrySheet = false
        refreshEntries()
    }
)
```

### Валидация

- Для новых записей: `!text.trim().isEmpty()` (проверяем что текст не пустой после trim)
- Для редактирования: `text.trim() != oldEntry.trim()` (проверяем что текст изменился)
- При пустом тексте отправлять событие `TextEntryEvent.Error` с сообщением `text_entry_empty_error`
- Проверять сеть перед отправкой (использовать ConnectivityManager)

### Обработка ошибок

- Отображать ошибки через Snackbar (на уровне `TextEntrySheetHost`)
- Использовать `errorReporter.handleError()` для централизованной обработки в ViewModel
- Логировать ошибки в консоль
- Валидация пустого текста отправляет событие `TextEntryEvent.Error` с локализованным сообщением
