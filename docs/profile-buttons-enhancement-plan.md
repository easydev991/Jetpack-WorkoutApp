# План доработки экрана профиля: добавление кнопок навигации

## Обзор

Добавить на экран профиля (`ProfileRootScreen`) кнопки навигации аналогично iOS-приложению. При нажатии на кнопки пока что должно происходить логирование в консоль (без реальной навигации). Все данные для кнопок уже доступны в модели `User`.

## Назначение

Сделать Android-версию экрана профиля консистентной с iOS-версией, добавив недостающие кнопки для навигации к функционалу приложения.

## Что сделано

- ✅ `ProfileRootScreen` - базовая реализация с отображением профиля и кнопкой выхода
- ✅ `ProfileViewModel` - загружает профиль пользователя, страну и город
- ✅ `User` модель - содержит все необходимые данные для кнопок:
  - `friendRequestCount` - количество заявок в друзья
  - `friendsCount` - количество друзей
  - `parksCount` (area_count) - количество площадок, где тренируется
  - `addedParks` - список добавленных площадок
  - `journalCount` - количество дневников
  - Вычисляемые свойства: `hasJournals`, `hasFriends`, `hasUsedParks`, `hasAddedParks`

## Требуемые кнопки (из iOS-приложения)

### iOS MainUserProfileScreen содержит

1. **Кнопка "Изменить профиль"**
2. **Кнопка "Друзья"** (с бейджем для заявок в друзья)
3. **Кнопка "Где тренируется"** (показывается если `hasUsedParks`)
4. **Кнопка "Добавленные площадки"** (динамическое название, показывается если `hasAddedParks`)
5. **Кнопка "Дневники"** (для главного пользователя всегда показывается)
6. **Кнопка "Черный список"** (показывается только если есть пользователи в черном списке)
7. **Кнопка "Выйти"** (уже реализована)
8. **Кнопка поиска пользователей** (иконка лупы в топ-баре справа)

## Примечания

### Ограничения текущей реализации

1. **Черный список**: В текущей версии `ProfileViewModel` нет доступа к списку пользователей в черном списке. Рекомендуется:
   - Вариант 1: Временно скрыть эту кнопку
   - Вариант 2: Добавить заглушку (показывать всегда, но с пустым списком)
   - Вариант 3: Добавить в `User` поле `blacklistedUsersCount` после реализации соответствующего API

2. **Бейдж для кнопки "Изменить профиль"**: В iOS-версии используется `user.badgeValue`, которое показывает количество непрочитанных обновлений. В Android-версии пока нет такого поля, поэтому можно показать бейдж если `friendRequestCount > 0`.

3. **Навигация**: Текущая задача - только отображение кнопок и логирование. Реальная навигация к экранам будет реализована в следующих этапах.
