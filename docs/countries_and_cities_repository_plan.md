# План реализации справочника стран и городов

## Обзор

Необходимо создать репозиторий для работы со справочником стран и городов, который:
- Загружает данные из локального JSON-файла `countries.json` на первой итерации
- Предоставляет возможность получения стран и городов по идентификаторам
- Поддерживает будущую интеграцию с API для обновления справочника
- Может быть использован из любой точки приложения

## Архитектурные решения

- **Гибридный подход**: чтение из локального JSON + возможность обновления с сервера
- **Отдельный репозиторий**: `CountriesRepository` для изоляции логики
- **Использование Flow**: реактивный доступ к данным
- **Кэширование в памяти**: однократная загрузка при первом обращении
- **Оптимизация поиска через Map**: O(1) поиск стран и городов по ID

## Выполненные этапы

### ✅ Этап 1: Domain Layer

Создан интерфейс `CountriesRepository` в `domain/repository/` и 4 use case:
- `GetCountriesUseCase` - получить список всех стран
- `GetCountryByIdUseCase` - получить страну по ID
- `GetCityByIdUseCase` - получить город по ID
- `GetCitiesByCountryUseCase` - получить города страны

### ✅ Этап 2: Data Layer

Реализован `CountriesRepositoryImpl` в `data/repository/`:
- Чтение из `assets/countries.json` с обработкой ошибок
- Кэширование в памяти через `MutableStateFlow`
- Оптимизация поиска через `citiesByIdMap` и `countriesByIdMap` (O(1))
- Метод `updateCountriesFromServer()` для будущей интеграции с API
- Логирование на русском языке, безопасное разворачивание опционалов

### ✅ Этап 3: Dependency Injection

Репозиторий внедрен в `AppContainer` через интерфейс с lazy-инициализацией. Доступен для всех ViewModels.

### ✅ Этап 4: Использование в приложении (частично)

- ✅ `ProfileViewModel` использует репозиторий для получения страны и города профиля
- ⚠️ Осталось интегрировать с экранами площадок и мероприятий

### ✅ Этап 5: Тестирование

Создан `CountriesRepositoryTest.kt` с 12 unit-тестами (MockK, без реальных сетевых запросов, мокирование Android Log).

## Будущее расширение

### 6.1. Обновление справочника с сервера

**Когда потребуется**:
- Автоматическое обновление при старте приложения (например, раз в неделю)
- Кнопка "Обновить справочник" в настройках
- Периодическое обновление в фоне

**Реализация**:
- Вызывать `countriesRepository.updateCountriesFromServer()` при необходимости
- Хранить дату последнего обновления в DataStore
- Показывать пользователю индикатор загрузки при обновлении

### 6.2. Оптимизация поиска городов по ID

✅ **Уже реализовано**: созданы `Map<String, City>` и `Map<String, Country>` для O(1) поиска стран и городов по ID вместо O(n).

### 6.3. Интеграция с экранами площадок и мероприятий

При создании следующих ViewModels необходимо интегрировать `CountriesRepository`:
- `ParkDetailViewModel` - отображение страны/города площадки
- `EventDetailViewModel` - отображение страны/города мероприятия
- `CreateParkViewModel` - выбор страны/города при создании площадки
- `CreateEventViewModel` - выбор страны/города при создании мероприятия
