# План реализации навигации в Jetpack-WorkoutApp

## Введение

Этот план описывает реализацию навигации в приложении Jetpack-WorkoutApp на основе официального примера Now in Android (android/nowinandroid).

## Основные принципы из Now in Android

### 1. Единый NavHost в приложении - ✅ РЕАЛИЗОВАНО

- Используется только один `NavHost` в главном экране приложения
- Все маршруты навигации определяются в одном месте
- Отсутствуют отдельные NavHost для каждой вкладки

### 2. AppState для управления состоянием навигации - ✅ РЕАЛИЗОВАНО

- Создается `AppState` класс (аналог `NiaAppState` в Now in Android)
- Управляет текущим пунктом назначения и верхнеуровневыми вкладками
- Хранит список верхнеуровневых назначений (top-level destinations)

### 3. Centralized Destinations - ✅ РЕАЛИЗОВАНО

- Все маршруты определены централизованно
- Используется sealed class для определения всех маршрутов
- Каждое назначение имеет уникальный маршрут

### 4. Navigation Options для вкладок - ✅ РЕАЛИЗОВАНО

- При навигации на верхнеуровневое назначение используются специальные опции:
  - `popUpTo(navController.graph.findStartDestination().id)` - очищает стек до стартовой точки
  - `saveState = true` - сохраняет состояние
  - `launchSingleTop = true` - избегает дублирования при повторном нажатии
  - `restoreState = true` - восстанавливает сохраненное состояние

## Текущее состояние Jetpack-WorkoutApp

### Проблемы (все решены в рамках реализации)

1. **Множественные NavHost** - ✅ РЕШЕНО
2. **Отсутствие AppState** - ✅ РЕШЕНО
3. **TabBarItem смешивает данные и UI** - ✅ РЕШЕНО
   - Класс `TabBarItem` содержал не только данные (route, titleId), но и UI-методы (Label(), Icon())

   - **Решение:** Класс удален (Январь 2026)

4. **Отсутствие централизованных маршрутов** - ✅ РЕШЕНО
